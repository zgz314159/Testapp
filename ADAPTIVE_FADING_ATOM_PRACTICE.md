# 自适应渐隐原子练习

## 1. 产品边界

本功能是原子题库的新增练习模式，不替换、不修改以下既有行为：

- 常规练习、考试、复盘、错题本、收藏题；
- 原子题库现有分数/标签筛选和全答模式；
- 原始题库 SQLite/JSON 的导入格式与内容。

原子题库继续作为只读知识源。自适应状态保存在应用数据库，以“题库 + 原题 + 空位”标识一个学习原子。

## 2. 核心流程

```text
原子题库
  → 核心池 / 覆盖抽查池
  → 到期、重要性、薄弱度选题
  → 选择填空 / 提示填空 / 完全填空
  → 现有 Practice Session
  → AnswerSubmitted Extension
  → 更新阶段、连续正确、遗忘次数与下次到期时间
```

同一个原子只有一份学习状态；选择、提示、完全填空只是动态呈现阶段，不生成三份互相独立的进度。

## 3. MVP 规则

### 3.1 双池

- 核心池：`数据/参数`、`强制性动词`，以及每条原文按权重排序的前三个原子。
- 覆盖抽查池：其他可评分原子。
- 抽查池答错后进入核心池；连续正确则维持低频抽查。
- 候选优先级：遗忘次数高 → 已学习且到期 → 未学习新原子，防止新知识持续挤占复习。

### 3.2 渐隐阶段

| 阶段 | 呈现 | 升级 |
|---|---|---|
| `CHOICE` | 同标签自动干扰项 | 跨次连续正确 2 次 |
| `HINTED` | 标签、字数、首字或单位提示 | 连续正确 2 次 |
| `RECALL` | 无提示完全填空 | 连续正确 3 次 |
| `MATURE` | 完全填空、低频到期 | 答错则降级 |

答错时降一级；选择阶段答错保持选择阶段并缩短重试间隔。

### 3.3 同条文避让

一次 Session 默认只抽取同一原题的一个空位。候选排序通过题库、原题、空位的稳定键完成，避免同一条文连续暴露答案。

### 3.4 判定

- MVP 沿用现有填空答案标准化与判定，避免改变普通练习语义；
- 数字型选择题的干扰项必须与正确答案单位一致；更严格的单位/否定词判定作为后续独立规则评估；
- 自动选择题只从其他条文获取同标签、非重复干扰项；数量不足时回退到提示填空。

## 4. 分层职责

| 模块 | 职责 |
|---|---|
| `:domain` | 阶段、池类型、状态模型与仓库接口 |
| `:data` | Room Entity/DAO、Repository、数据库迁移 |
| `:core` | 继续提供原子答案解析和派生题号能力 |
| `:feature-practice` | 候选构建、题目呈现、Session、进度 Extension |
| `:app` | Session Creator 注册、路由、首页入口与 Hilt Set 注册 |

## 5. 调度边界

MVP 使用可解释的固定间隔：答错短期重试；选择、提示、回忆、熟练阶段逐步延长。后续可在不改变 Session/UI 的前提下，将仓库中的 `difficulty/stability/dueAt` 接入 FSRS 类个性化调度。

## 6. 验收条件

- 普通练习和考试的 Kind、路由及加载逻辑保持原样；
- 仅 `.sqlite` / `.db` 原子题库入口显示新模式；
- 新模式每条原题每轮最多生成一个题目；
- 完成答题后自适应状态可跨进程保留；
- 再次进入时题型可按历史从选择升级为提示/完全填空；
- `ktlintCheck`、相关单测、架构测试和 APK 构建通过。

## 7. 当前上传题库核对（只读）

- 189 条原文，3,351 个可评分原子；每条原文 3–130 个，平均约 17.73 个；
- 主要标签为固定术语 2,735、情态动词 367、动词/谓语 88、强制性动词 61、数据/参数 37；
- 权重范围 3–10，足以执行“每条前三 + 强制词/参数必入核心池”；
- 按当前规则初始核心池约 658 个（19.6%），其余进入低频探测池，避免 3,351 个原子等量复习；
- 因单条原文原子很多，同轮只出一个原子的避让规则对这份题库尤其必要。
